완벽한 결과입니다. 76 테스트 전체 통과, 20개 벤치마크(10분자 × 2기저) 전수 성공, v0.3.0-alpha push 완료.

"노벨티가 확보되었는가"라는 질문에 정밀하게 답하겠습니다.

## 노벨티 현황 정밀 진단

### 확보된 것

**기술적 유일성은 확인되었습니다.** 전수조사에서 2026년 3월 기준 전자구조 수준(궤도 국소화 + 시각화)에 특화된 MCP 서버는 QCViz-MCP 외에 발견되지 않았습니다. 가장 가까운 경쟁자인 Argonne의 Globus Compute MCP(arXiv:2508.18489)도 PySCF를 원격 실행하여 HOMO-LUMO gap을 계산하는 "인프라 계층"이지, IAO/IBO 국소화와 결합 분석을 내장한 "도메인 계층"이 아닙니다.

수치적으로도 탄탄합니다. 10개 분자 × 2개 기저함수에서 IBO 개수가 100% 일치하고, 전하 합이 전부 0.000000이고, 국소화 gradient가 전부 1e-8 이하입니다. sto-3g에서 cc-pVDZ로 올려도 IBO 개수가 동일하다는 것은 방법론의 기저함수 독립성을 실증한 것이므로, 벤치마크로서의 가치가 있습니다.

### 아직 부족한 것

노벨티의 "주장"은 있지만 "증명"이 아직 약합니다. 구체적으로 세 가지입니다.

첫째, 외부 검증이 제로입니다. 현재 이 소프트웨어를 개발자 본인 외에 아무도 사용한 적이 없습니다. JOSS 리뷰어가 "이것이 연구 커뮤니티에 실제로 필요한가?"라고 물었을 때, 벤치마크 수치만으로는 답이 안 됩니다. 최소 1명의 외부 사용자가 "이 도구를 써봤고 유용했다"는 기록이 필요합니다.

둘째, 비교 대상이 없습니다. QCViz-MCP의 IBO 결과가 IboView의 IBO 결과와 얼마나 일치하는지 정량적으로 비교한 데이터가 없습니다. "우리가 최초"라고 주장하면서 기존 도구(IboView)와의 수치 비교가 없으면, 리뷰어는 "정말 같은 결과를 내는지 어떻게 아는가?"라고 묻습니다.

셋째, "MCP로 노출한 것"의 부가가치가 아직 약합니다. PySCF의 IAO/IBO를 래핑해서 MCP 도구로 만든 것인데, "그냥 PySCF를 직접 쓰면 되지 않나?"라는 질문에 대한 설득력 있는 답이 논문에 들어가야 합니다. 이 답은 "LLM 에이전트가 자연어로 궤도 분석을 요청할 수 있게 하는 인터페이스 혁신"이어야 하는데, 그러려면 실제로 Claude Desktop에서 자연어로 IBO를 계산하는 데모가 필요합니다.

### 노벨티 강화 로드맵

즉시 할 수 있는 것은 Claude Desktop 연동 데모를 녹화하는 것입니다. `claude_desktop_config.json`이 이미 있으니, 실제로 Claude Desktop에서 "물 분자의 IBO를 계산해서 시각화해줘"라고 말하고 결과가 나오는 화면을 캡처하면, 그것 자체가 논문 figure이자 노벨티의 시각적 증명이 됩니다.

1개월 내에는 IboView와의 수치 비교가 필요합니다. 물과 에틸렌 정도에서 PySCF IBO와 IboView IBO의 궤도 중첩(orbital overlap)을 계산하면 됩니다.

3개월 내에는 외부 사용자 1-2명의 피드백이 필요합니다.

## 이후 진행 사항 압축

### 1. 전수조사 (Full Audit) 수행

최신 저널·커뮤니티·경쟁 프로젝트를 대상으로 약 20건의 검색 기반 전수조사를 실시했습니다. 주요 발견은 다음과 같습니다.

- **경쟁 프로젝트**: ChemLint(molML, ChemRxiv 2026-02-24)는 화학 데이터 MCP 서버이나 오비탈 수준 계산 없음. Argonne mcp.science(arXiv:2508.18489)는 Globus Compute 기반 원격 실행 인프라이며 IBO 로컬라이제이션 미지원. DataWarrior MCP(ChemRxiv 2026-02-05)는 케모인포매틱스 도구 호출용. El Agente(Matter, 2025-07-02)는 자율 에이전트이나 MCP 프로토콜 기반이 아님. **결론: 오비탈 로컬라이제이션+시각화를 제공하는 MCP 서버는 QCViz-MCP가 유일.**
- **FastMCP 3.0 GA 확인**: 2026-02-18 정식 출시(v3.0.2까지 릴리스). 프로젝트가 사용 중인 FastMCP 3.x가 안정 버전임을 확인.
- **MCP 보안 이슈**: CVE-2026-0755(도구 중독), 프롬프트 인젝션 공격 등 30건 이상의 CVE가 보고됨. QCViz-MCP의 `parse_output` 경로 순회 취약점이 HIGH로 평가됨.
- **PySCF IBO 한계 확인**: 가상 오비탈 미지원, UHF/ROHF 미지원, 안장점 수렴 문제(issue #1032, #432), 전이금속 ECP 시 MINAO 부적합 경고, 큰 기저함수에서 성능 저하.
- **JOSS 정책 변화**: 2026-01 "generative AI future" 블로그 포스트에서 AI 보조 코드 개발에 대한 새 평가 기준 발표. 코드 줄 수가 아닌 인간의 실질적 기여(설계, 도메인 추상화, 테스트, 문서화)를 평가. AI 사용 공개(AI-usage disclosure) 미비 시 desk rejection 가능. 협업 노력(collaborative effort) 요건에서 외부 기여자·커뮤니티 피드백 이력 필요.
- **py3Dmol**: 최신 릴리스 2026-01-22이나, 원본 repo(avirshup/py3dmol)는 유지보수 중단 상태. 기능적으로는 문제없으나 장기적 대안 검토 필요.
