## Phase ζ에서 한 일

### ζ-1: 전이금속 IAO/IBO 지원

PySCF의 IAO 계산은 기본 참조 기저로 `minao`를 사용하는데, ECP(유효 내핵 퍼텐셜)를 쓰는 전이금속에서는 MINAO 기저와 불일치가 발생한다. 이를 해결하기 위해 `pyscf_backend.py`에 `_resolve_minao(mol, minao)` 헬퍼 메서드를 추가했다. 이 메서드는 `mol.has_ecp()`를 호출해서 ECP 사용을 감지하고, 감지되면 자동으로 `minao='sto-3g'`로 폴백하면서 경고 메시지를 반환한다. `compute_iao()`에도 `minao` 파라미터를 추가하여 사용자가 명시적으로 참조 기저를 지정할 수 있게 했다.

벤치마크 분자로 Fe(CO)₅(iron pentacarbonyl, def2-SVP)와 TiCl₄(titanium tetrachloride, def2-SVP), 그리고 오픈셸 분자 NO 라디칼(STO-3G, spin=1)을 `benchmark/molecules.py`에 등록했다.

테스트 6개를 작성했다: ECP 감지(lanl2dz+ecp 명시), all-electron 기저 확인(def2-SVP), ECP 자동 폴백, ECP 없을 때 minao 유지, 명시적 override, water에 minao='sto-3g' 전달 시 IBO 5개 확인.

**실행 중 발견된 문제와 수정**: PySCF에서 `basis="lanl2dz"`만 지정하면 ECP가 자동 적용되지 않아 `mol.has_ecp()`가 빈 dict `{}`(falsy)를 반환했다. 테스트에 `ecp="lanl2dz"`를 명시적으로 추가하고, `_resolve_minao`의 ECP 감지 로직도 `len(ecp_result) > 0` 체크로 보강했다.

### ζ-2: UHF/ROHF IBO 완전 지원

기존 코드는 RHF(Restricted Hartree-Fock)만 지원했는데, UHF(Unrestricted HF)에서는 `mf.mo_coeff`가 alpha/beta 스핀 채널로 분리된다. 기존 RHF 경로를 수정하지 않고, 새 메서드 4개를 병렬로 추가했다.

`compute_scf_flexible(spin=N)`은 spin>0이면 자동으로 UHF를 선택한다. `compute_iao_uhf()`는 alpha/beta 점유 MO를 각각 IAO로 변환한다. `compute_ibo_uhf()`는 alpha/beta를 각각 IBO 로컬라이즈한다. `compute_uhf_charges()`는 UHF 밀도 행렬로 Mulliken 전하를 계산한다.

테스트 6개를 작성했다: O₂ 삼중항 SCF 수렴, O₂ IAO alpha/beta 분리, O₂ IBO에서 alpha > beta(unpaired 전자 반영), NO 라디칼에서 alpha = beta + 1, UHF 전하 합 ≈ 0, RHF 경로 불변 확인.

**실행 중 발견된 문제와 수정**: PySCF UHF의 `mo_coeff`가 tuple이 아닌 shape `(2, nao, nmo)`의 3D ndarray로 반환되었다. `isinstance(mf.mo_coeff, tuple)` 체크가 실패했고, `make_rdm1()`도 `(2, nao, nao)` 3D ndarray를 반환해서 einsum subscript `"ij,ji->i"`와 차원이 불일치했다. `_unpack_uhf()` 정적 메서드를 추가하여 tuple이든 3D ndarray든 통일 처리하고, `compute_uhf_charges()`에도 `dm.ndim == 3` 분기를 추가했다. 테스트의 타입 체크도 `mo_coeff.ndim == 3`으로 수정했다.

### ζ-3: 헤드리스 PNG 내보내기

`src/qcviz_mcp/renderers/` 패키지를 새로 만들고 `png_exporter.py`를 작성했다. Playwright + Chromium + SwiftShader를 사용하여 py3Dmol HTML을 headless로 PNG 캡처한다. `html_to_png()` (async) 와 `html_to_png_sync()` (동기 래퍼) 두 함수를 제공한다. `device_scale_factor=2`로 Retina 품질(실효 2배 해상도) PNG를 생성한다.

Playwright는 선택적 의존성(`pyproject.toml`에 `png = ["playwright>=1.40"]` 추가)으로 처리하여, 미설치 환경에서는 PNG 기능만 비활성화되고 나머지 도구는 정상 동작한다.

테스트 5+1개를 작성했다: 기본 PNG 캡처, 크기 지정, 자동 경로 생성, 동기 래퍼, graceful degradation(import만 확인). 현재 환경에 Playwright가 미설치되어 4개는 skip, graceful degradation 1개는 passed.
